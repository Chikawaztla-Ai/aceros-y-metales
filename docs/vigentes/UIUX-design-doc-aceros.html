01 — Principios de Diseño

5 reglas que guían toda decisión visual

1. Industrial, no decorativo

El diseño transmite confianza técnica. Nada de gradientes, sombras dramáticas ni ilustraciones genéricas. Los datos hablan solos.

2. Dato visible > clic extra

Si un precio, stock o peso se puede mostrar sin abrir un modal, se muestra. Menos clics = más conversiones.

3. Mobile-capable, desktop-first

Los compradores B2B usan escritorio. El sitio es responsive, pero la experiencia de escritorio se prioriza.

4. Confianza por transparencia

Mostrar stock real, pesos calculados, equivalencias internacionales. El comprador técnico valora la exactitud.

5. Conversión sin fricción

Cada pantalla tiene un CTA claro. Nunca más de 2 opciones: "Agregar al carrito" o "Solicitar cotización".

Estados vacíos y de error

Carrito vacío

Ícono shopping_cart grande + "Tu carrito está vacío" + CTA "Explorar catálogo". Sin tabla ni resumen.

Sin resultados de búsqueda

"No encontramos materiales para [query]" + sugerencias: "¿Quisiste decir 4140?" + "Solicitar cotización personalizada".

Error de pago

Ícono error rojo + "El pago no pudo procesarse" + opciones: "Reintentar" o "Pagar por transferencia". NUNCA un error genérico.

Producto sin stock

Badge "Agotado" en rojo. Botón cambia a "Notificarme cuando esté disponible" (captura email). Quitar botón de carrito.

Regla general: Animaciones solo para feedback funcional, nunca decorativas. Duración máxima 300ms para interacciones directas. Respetar prefers-reduced-motion — desactivar animaciones si el usuario lo solicita.

06 — Accesibilidad (WCAG 2.1 AA)

Requisitos mínimos

Contraste

Texto normal: ratio ≥ 4.5:1. Texto grande (≥18px bold): ratio ≥ 3:1. Verificar con el fondo Steel Blue + naranja. El sistema ya cumple — validado en Stitch.

Navegación por teclado

Tab order lógico en formularios. Focus visible (ring-2) en todos los interactivos. Skip to content link oculto.

Screen readers

Alt text en imágenes de productos. aria-label en botones de ícono (carrito, búsqueda). role="alert" en toasts y errores.

Formularios

Labels visibles en todos los inputs. Errores asociados con aria-describedby. Autocompletado habilitado (autocomplete="email" etc).

Touch targets

Mínimo 44x44px en mobile para botones y links. Spacing suficiente entre targets adyacentes (≥8px).

Semántica HTML

nav, main, article, aside, footer. Headings en orden (h1 → h2 → h3). Una sola h1 por página.

Regla del "doble CTA"

Cada pantalla de producto ofrece exactamente 2 caminos: "Comprar ahora" (carrito → checkout → pago) y "Cotizar" (formulario → email/WhatsApp). El comprador B2B que necesita aprobación interna siempre tiene la opción de cotización sin tener que abandonar el flujo.
