MVP — 6 Semanas

Semana por semana con tareas y entregables

Cada semana tiene un entregable verificable. El cliente puede ver el avance en el preview de Vercel.

SEMANA 1Setup y Fundamentos
SETUP
Init Next.js 15 con TypeScript, Tailwind, shadcn/ui. Estructura de directorios del PRD.
3h
Crear proyecto Supabase — ejecutar migrations SQL del Backend Schema.
4h
Configurar Supabase Auth — email/password, roles admin/customer, middleware de Next.js.
3h
Deploy inicial en Vercel — conectar repo GitHub, configurar dominio + Cloudflare DNS.
2h
Configurar variables de entorno en Vercel (dev/preview/prod) según .env.local del PRD.
1h
Componentes base — Header, Footer, Layout público, Layout admin, Layout portal.
4h
Seed data — cargar 10 categorías + 20-30 productos del catálogo aprobado en Supabase.
3h
Entregable semana 1:

Sitio en vivo en acerosymetalesurgentes.com con header/footer, base de datos con datos reales y auth funcionando.

SEMANA 2Catálogo y Producto
FRONTEND
Homepage — hero, categorías, productos destacados, barra de confianza. ISR 1h.
6h
Catálogo — grid de productos, filtros (categoría, grado, unidad), paginación, búsqueda SQL.
8h
Ficha de producto — galería, specs, equivalencias, descarga PDF/DWG. ISR 30min.
6h
Calculadora de peso — useWeightCalc() hook + componente reutilizable. Unit tests.
4h
SEO metadata — generateMetadata() dinámico en producto y catálogo. JSON-LD Product.
2h
Entregable semana 2:

Catálogo navegable con filtros, fichas de producto con calculadora de peso funcionando. El cliente puede navegar el catálogo real.

SEMANA 3Carrito, Checkout y Pagos
BACKEND
Zustand store — carrito con persistencia localStorage, acciones add/remove/update/clear.
3h
Página de carrito — lista de items, ajustar cantidades, peso total, resumen.
4h
Checkout — formulario 3 secciones (facturación/envío/pago), validación Zod.
6h
Mercado Pago — POST /api/pagos/crear + webhook + verificación de firma.
6h
Transferencia bancaria — flujo pending_transfer, datos bancarios, confirmación admin.
4h
Edge Function — cancel-expired-transfers (cron 1h, timeout 48h).
2h
Confirmación de pedido — pantalla + email con Resend + react-email template.
3h
Entregable semana 3:

Flujo de compra completo end-to-end: agregar al carrito → checkout → pago (MP o transferencia) → confirmación. Probado con tarjeta de test de MP.

SEMANA 4Cotizaciones, Wishlist, Promos
BACKEND
Cotizaciones — formulario multi-material, upload PDF/DWG a Storage, generar PDF con @react-pdf.
6h
Email cotización — al admin con PDF adjunto + link WhatsApp. Al cliente con confirmación.
3h
Wishlist — API routes + UI (corazón en cards de producto + página de wishlist en portal).
3h
Promociones — CRUD admin + validación de cupones en checkout + descuento aplicado.
4h
Buscador inteligente — OpenAI embeddings, indexar productos, match_products(), componente SmartSearch.
6h
Botón carrito→cotización — convertir items del carrito en cotización formal.
2h
Entregable semana 4:

Cotizaciones funcionando end-to-end. Buscador IA funcionando ("varilla 3/8" devuelve resultados relevantes). Promociones aplicables en checkout.

SEMANA 5Dashboard Admin + Portal Cliente
FRONTEND
Dashboard — KPIs (ventas, pedidos, cotizaciones, clientes nuevos), gráfica de ventas.
4h
Admin pedidos — lista con filtros, detalle, cambiar status, confirmar transferencia.
5h
Admin inventario — tabla de stock, edición manual, alertas de stock bajo, importar CSV.
4h
Admin cotizaciones — lista pendientes, ver detalle + PDF/DWG, responder.
3h
Admin clientes — lista, cambiar tier, ver historial de pedidos.
3h
Portal cliente — mis pedidos, mis cotizaciones, datos de empresa, nivel y descuento.
5h
Entregable semana 5:

Dashboard admin y portal del cliente completos. El admin puede gestionar pedidos, inventario y cotizaciones. El cliente puede ver su historial.

SEMANA 6SEO, Blog, QA y Go-Live
QA + LAUNCH
Blog — listado + artículo individual + admin editor + tabla de contenidos + CTA conversión.
5h
3 artículos iniciales — redactar y publicar 3 artículos SEO (4140, 6061, inoxidable 304).
3h
SEO técnico — sitemap.ts dinámico, robots.txt, JSON-LD, OpenGraph images.
2h
Analytics — GA4 + Meta Pixel + Microsoft Clarity + eventos de conversión.
2h
Google Business Profile — crear/optimizar perfil, fotos, horario, link al sitio.
2h
Search Console — verificar propiedad, enviar sitemap, monitorear indexación.
1h
QA completo — 10 flujos del checklist del PRD, Lighthouse, responsive, cross-browser.
6h
Fixes finales — correcciones de QA, ajustes de diseño, polish.
4h
Go-live — merge final a main, verificar producción, notificar al cliente.
1h
🚀 GO-LIVE

Sitio en producción en acerosymetalesurgentes.com. Blog con 3 artículos. Analytics configurado. Google Business activo. Todos los flujos probados y funcionando.
